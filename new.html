Digital Forensics Labs – NWIT 263

A collection of hands-on labs covering forensic imaging, hashing, metadata analysis, steganography, and Windows artifacts.

Lab 02 – Kali Linux Installation (9/11/2025)

Objective: Install Kali Linux in a virtual environment and verify its functionality.

Steps/Details:

  • Installed Kali Linux ISO using VMware Fusion.
  • Configured VM: 4GB RAM, 20GB disk, Debian (64-bit).
  • Verified system via uname -a and lsb_release -a.

Conclusion: Successfully created a working Kali Linux environment for forensic use.

Lab 03 – Hashing Lab (Windows & Linux) (9/19/2025)

Objective: Demonstrate how file hashing ensures data integrity and authenticity.

  • Used CertUtil in Windows and sha256sum/md5sum in Linux.
  • Generated/comparing MD5, SHA1, SHA256 hashes.
  • Confirmed altered files produced different hashes.

Conclusion: Showed that hashing is reliable for tamper detection in digital forensics.

Lab 04 – FTK Imager: Create, Mount, and Dismount (9/19/2025)

Objective: Use FTK Imager to create and verify a forensic disk image.

  • Created E01 forensic image with MD5/SHA1 verification.
  • Mounted image in read-only mode (Drive F:).
  • Verified integrity and dismounted image safely.

Conclusion: Demonstrated proper forensic imaging and verification workflow.

Lab 05 – Enabling & Disabling USB Write Protection (9/26/2025)

Objective: Simulate a software-based write blocker using Windows Registry.

  • Enabled write protection via StorageDevicePolicies key.
  • Tested write failure; reverted change to restore write access.

Conclusion: Demonstrated how software write blockers preserve evidence integrity.

Lab 06 – Extracting and Mapping GPS Data (10/03/2025)

Objective: Extract GPS metadata and map coordinates using ExifTool.

  • Used ExifTool to extract GPS coordinates from image.
  • Converted DMS to decimal and plotted in Google Maps.
  • Removed metadata to demonstrate privacy protection.

Conclusion: Highlighted forensic value and privacy risks of photo metadata.

Lab 07 – File Hashing with Febooti Hash & CRC (10/16/2025)

Objective: Verify file integrity using Febooti Hash & CRC utility.

  • Generated MD5 and SHA-256 hashes before and after file edits.
  • Observed avalanche effect when data was modified.

Conclusion: Reinforced importance of hashing for digital evidence validation.

Lab 08 – Extracting Metadata from MS Office Files (10/23/2025)

Objective: Examine file metadata such as creation/modification timestamps.

  • Viewed metadata through file properties.
  • Compared Date Created vs. Date Modified.
  • Explained forensic significance of file timestamps.

Conclusion: Metadata reveals timeline and authorship for forensic reconstruction.

Lab 09 – Windows Forensic Artifacts (10/23/2025)

Objective: Identify persistence and activity artifacts within Windows systems.

  • Examined Registry Run keys, Prefetch, and Recent Files.
  • Detected traces of suspicious execution and persistence.

Conclusion: Demonstrated artifact analysis for attacker activity reconstruction.

Lab 10 – Hard Links vs. Soft Links (10/31/2025)

Objective: Compare file system link behaviors and forensic implications.

  • Created and analyzed hard links and symbolic links in Linux.
  • Observed inode sharing and persistence after deletion.

Conclusion: Explained link behavior relevance in forensic data recovery.

Lab 11 – Hidden Message Extraction with Steghide (10/31/2025)

Objective: Use steganography to hide and recover data within images.

  • Embedded text file inside a JPEG using Steghide.
  • Verified differences via hashing and metadata analysis.
  • Extracted hidden message successfully.

Conclusion: Demonstrated covert data hiding and forensic detection techniques.
